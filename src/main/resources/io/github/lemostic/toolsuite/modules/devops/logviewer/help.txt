服务器日志查看器 - 使用指南

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
一、快速开始
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 添加服务器
   • 点击顶部工具栏的"管理"按钮
   • 在服务器管理对话框中点击"添加"
   • 填写服务器信息：
     - 服务器名称：便于识别的名称（如：生产服务器）
     - 主机地址：服务器IP或域名
     - SSH端口：默认为22
     - 用户名：SSH登录用户名
     - 密码或私钥：选择认证方式并填写

2. 配置日志目录
   • 在添加/编辑服务器时，配置日志目录
   • 每个目录需要设置：
     - 目录名称：显示在Tab上的名称
     - 目录路径：服务器上的绝对路径（如：/var/log/app）
     - 文件过滤：文件匹配模式（如：*.log）

3. 连接服务器
   • 从下拉框选择要连接的服务器
   • 点击"连接"按钮
   • 连接成功后，会自动加载配置的日志目录

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
二、查看日志
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 切换目录
   • 使用Tab页切换不同的日志目录
   • 每个目录显示独立的文件列表

2. 打开日志文件
   • 在文件列表中双击要查看的文件
   • 或使用"加载行数"下拉框选择要加载的行数后双击

3. 行数选择
   • 尾200行：加载文件最后200行（默认）
   • 尾500行：加载文件最后500行
   • 尾1000行：加载文件最后1000行
   • 头200行：从文件开头加载200行
   • 头500行：从文件开头加载500行
   • 头1000行：从文件开头加载1000行

4. 搜索日志
   • 在搜索框输入关键词
   • 点击"搜索"按钮或按回车键
   • 自动高亮并定位到第一个匹配项
   • 继续搜索会定位到下一个匹配项

5. 复制日志
   • 选中要复制的文本，点击"复制"按钮
   • 或直接点击"复制"按钮复制全部内容

6. 刷新文件列表
   • 点击"刷新"按钮重新加载当前目录的文件列表

三、快捷键
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

• Ctrl+F：聚焦到搜索框
• Enter：执行搜索
• F5：刷新文件列表

四、注意事项
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 服务器要求
   • 服务器需要开启SSH服务
   • 用户需要有日志目录的读取权限
   • 服务器需要支持 find 命令

2. 性能建议
   • 对于大日志文件，建议只加载尾部行数
   • 避免同时打开多个大文件

3. 安全性
   • 服务器密码在本地加密存储
   • 建议使用私钥认证提高安全性
   • 私钥密码可选，如有会加密存储

五、故障排除
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 连接失败
   • 检查服务器地址和端口是否正确
   • 确认用户名和密码/私钥正确
   • 检查服务器SSH服务是否运行
   • 确认防火墙允许SSH连接

2. 无法读取文件列表
   • 确认日志目录路径正确
   • 确认用户有目录读取权限
   • 检查文件过滤模式是否正确

3. 读取日志内容失败
   • 确认文件存在且可读
   • 检查文件是否过大导致超时
