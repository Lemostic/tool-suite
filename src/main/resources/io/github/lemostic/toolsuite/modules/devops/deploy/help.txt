================================================================================
                            应用部署工具 - 使用说明
================================================================================

【功能概述】
应用部署工具是一个用于快速部署应用程序到远程服务器的图形化工具。支持多服务器
并行部署、自动备份、权限设置等功能。

【主要特性】
✓ 多服务器配置管理
✓ 可视化服务器选择
✓ 程序包拖放上传
✓ 灵活备份策略（全部/选择性/模式匹配）
✓ 文件权限自动设置
✓ 实时部署日志输出
✓ 失败自动回滚

================================================================================
【使用步骤】
================================================================================

1. 配置服务器
   - 点击工具栏"添加服务器"按钮
   - 填写服务器信息：
     * 名称：便于识别的服务器名称
     * 主机：IP地址或域名
     * 端口：SSH端口（默认22）
     * 用户名/密码：SSH登录凭据
     * 程序目录：应用部署的目标目录
     * 备份目录：备份文件存储目录
     * Bin目录：启停脚本所在目录（相对程序目录）
     * 启停脚本：服务启动和停止的脚本文件名

2. 选择目标服务器
   - 在左侧服务器列表中勾选要部署的服务器
   - 可以勾选多台服务器进行并行部署

3. 选择程序包
   - 方式一：直接将文件拖放到拖放区域
   - 方式二：点击"选择文件"按钮浏览选择
   - 支持格式：.zip, .tar.gz, .tgz, .jar, .war

4. 配置部署选项
   【备份选项】
   - 备份所有文件：备份程序目录下的所有内容
   - 选择性备份：仅备份指定的文件或目录
   - 模式匹配：按文件名模式备份（如 *.jar）
   - 删除原文件：备份成功后删除原程序文件

   【上传选项】
   - 更新文件权限：上传后设置文件权限
   - 权限值：如 755（rwxr-xr-x）
   - 所有者/用户组：设置文件所属用户和组
   - 保留时间戳：保留文件的原始修改时间

5. 开始部署
   - 点击"开始部署"按钮
   - 确认部署信息后点击"确定"
   - 在下方日志区域查看各服务器的部署进度

================================================================================
【部署流程】
================================================================================

每个服务器的部署按以下步骤顺序执行：

1. 连接服务器
   - 建立SSH连接
   - 验证凭据

2. 停止服务
   - 执行停止脚本
   - 等待服务停止

3. 备份原程序
   - 根据备份选项执行备份
   - 备份文件按时间戳命名

4. 上传新程序
   - 上传程序包到服务器
   - 自动解压压缩包
   - 设置文件权限和所有者

5. 启动服务
   - 执行启动脚本
   - 检查服务状态

================================================================================
【故障处理】
================================================================================

1. 连接失败
   - 检查主机地址和端口是否正确
   - 确认用户名和密码无误
   - 确保SSH服务正常运行
   - 检查防火墙设置

2. 备份失败
   - 确认备份目录存在且有写入权限
   - 检查磁盘空间是否充足

3. 上传失败
   - 检查程序目录权限
   - 确认磁盘空间充足
   - 检查网络连接稳定性

4. 服务启动失败
   - 查看日志了解具体错误
   - 检查配置文件是否正确
   - 确认端口未被占用

5. 部署失败回滚
   - 部署过程中任何步骤失败都会自动触发回滚
   - 已停止的服务会尝试重新启动
   - 可手动从备份目录恢复程序

================================================================================
【安全建议】
================================================================================

1. 密码安全
   - 建议使用强密码
   - 定期更换服务器密码
   - 考虑使用密钥认证代替密码

2. 备份策略
   - 定期清理过期备份文件
   - 重要部署前手动备份
   - 验证备份文件完整性

3. 权限设置
   - 遵循最小权限原则
   - 避免使用root账户部署
   - 正确设置文件权限防止未授权访问

================================================================================
【快捷键】
================================================================================

F5          刷新服务器列表
F1          打开帮助
